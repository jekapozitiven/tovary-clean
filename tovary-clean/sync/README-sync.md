# Движок синхронизации товаров

Переносит поля товаров с **Black Street** на **Bonna-shop** и **Core22** через
Prom API. Работает сам, по расписанию, внутри GitHub Actions. Сервер не нужен,
всё бесплатно.

## Что нужно один раз настроить

### 1. Создать API-токены (3 штуки)

В каждом магазине по очереди: кабинет Prom → **Настройки** → **Управление
API-токенами** → **Создать токен**. Скопируй значение.

Токены — это как пароли. **Никому не отправляй их, не вставляй в чат и не клади
в файлы репозитория.** Дальше они пойдут в защищённое хранилище GitHub.

### 2. Положить токены в GitHub Secrets

Репозиторий → **Settings** → слева **Secrets and variables** → **Actions** →
кнопка **New repository secret**. Создай три штуки (имя = точно как ниже,
значение = токен нужного магазина):

- `TOKEN_BLACKSTREET`
- `TOKEN_BONNA`
- `TOKEN_CORE22`

После сохранения GitHub прячет их — даже ты больше не увидишь значение. Это норма.

### 3. Положить config.json

Открой на портале страницу **Актуализация цен**, отметь галочками нужные поля,
нажми **Скачать config.json** и загрузи этот файл в папку `sync/` репозитория
(рядом с `sync.py`). Пример уже лежит в `config.example.json`.

### 4. Включить Actions

Вкладка **Actions** в репозитории → если попросит подтвердить запуск workflow —
согласись (**I understand my workflows, go ahead and enable them**).

## Первый запуск — пробный (ничего не меняет)

1. Вкладка **Actions** → слева **Синхронизация товаров** → кнопка **Run
   workflow**.
2. Поле dry_run оставь **1**.
3. Запусти и открой лог. Движок покажет, сколько товаров нашёл и что собирается
   изменить — но ничего не отправит.

Если в логе видно товары и разумные изменения — всё подключено верно.

## Боевой режим

По расписанию (каждые ~10 минут) движок запускается сам уже в боевом режиме.
Ничего нажимать не надо. Журнал всех запусков — во вкладке **Actions**.

## Что переносится быстро, а что медленно

- **Быстро (~10 минут), через правку по API:** цена, наличие, количество,
  скидка, скрыть/показать.
- **Медленнее, через импорт Prom:** название, артикул, описание, фото. Пока эти
  поля в движке помечены как «фаза 2» и не отправляются — включим после первого
  успешного пробного запуска, когда убедимся, как связаны товары между
  магазинами.

## Важно про связь товаров

Движок находит «тот же самый» товар в другом магазине по стабильному
**external_id** (внутренний код), а НЕ по артикулу. Это специально: когда ты
поменяешь артикул, связь не потеряется. Поэтому нужно, чтобы у товаров на
приёмниках был проставлен external_id, совпадающий с Black Street. Если это не
так — сначала сделаем разовую привязку. Это тот вопрос, который надо подтвердить
до боевого включения.
