# Товары — портал приложений

Тёмный портал-каталог для внутренних приложений по товарам. Работает как обычный
сайт: открывается на компьютере, планшете и телефоне, вёрстка подстраивается под
любой экран. Каждое приложение имеет сверху кнопку **«Главная»**, а выбранные
настройки не сбрасываются при обновлении страницы.

## Что внутри

```
tovary-portal/
├── index.html              ← главная страница (меню приложений)
├── assets/
│   └── portal.css          ← общие стили (цвета, шрифты, карточки)
├── apps/
│   └── example/
│       └── index.html      ← приложение-образец (шаблон для новых)
├── .nojekyll               ← служебный файл для GitHub Pages
└── README.md               ← этот файл
```

## Как выложить на GitHub Pages (без командной строки)

1. Зайдите на **github.com** и создайте новый репозиторий (кнопка **New**).
   Имя, например, `tovary`. Тип — **Public**.
2. На странице репозитория нажмите **Add file → Upload files** и перетащите туда
   **всё содержимое** папки `tovary-portal` (файлы `index.html`, папки `assets`
   и `apps`, файл `.nojekyll`). Нажмите **Commit changes**.
3. Откройте **Settings → Pages**. В разделе *Build and deployment* выберите
   Source — **Deploy from a branch**, ветку **main**, папку **/(root)** и нажмите
   **Save**.
4. Через 1–2 минуты вверху появится ссылка вида
   `https://ВАШ_ЛОГИН.github.io/tovary/` — это адрес вашего портала.

Эту ссылку можно открыть **на телефоне** в любом браузере — всё отобразится
корректно. Чтобы был как иконка приложения, добавьте страницу на домашний экран
(в браузере: «Поделиться» → «На экран Домой»).

> Если предпочитаете git: `git init`, `git add .`, `git commit -m "portal"`,
> затем `git remote add origin ...` и `git push`. Репозиторий уже
> проинициализирован — можно сразу добавлять `remote` и пушить.

## Как добавить новое приложение

1. Создайте папку внутри `apps/`, например `apps/dosales/`, и положите туда
   `index.html` вашего приложения. Проще всего **скопировать** `apps/example/`
   и переделать под себя.
2. В новом приложении оставьте сверху блок шапки с кнопкой «Главная» — он уже
   есть в образце и помечен комментарием.
3. Откройте `index.html` в корне и допишите приложение в массив `APPS`:

   ```js
   {
     icon:   '💰',
     title:  'Допродажи',
     desc:   'Короткое описание.',
     href:   'apps/dosales/index.html',
     status: 'active'          // 'active' — кликабельно, 'soon' — серым «Скоро»
   }
   ```

Карточка появится на главной автоматически.

## Чтобы настройки не сбрасывались при обновлении

Смотрите приложение-образец `apps/example/index.html`. Все значения (вкладка,
переключатели, текст) хранятся в памяти браузера через `localStorage`. При
обновлении страницы (F5) состояние восстанавливается, и с приложения не
выкидывает. В своём приложении используйте тот же подход и **свой уникальный
ключ**: в образце это `const STORAGE_KEY = 'tovary:example';` — поменяйте
`example` на имя вашего приложения.

## Где менять текст

- Название и подпись портала — в `index.html` (блок `brand`) и в шапке каждого
  приложения.
- Цвета и шрифты — в начале `assets/portal.css` (раздел `:root`), например
  акцентный цвет `--accent`.
- Если нужен украинский интерфейс — текст в карточках и заголовках правится
  прямо в этих же файлах (например `ТОВАРЫ` → `ТОВАРИ`, `Активно` → `Активно`).
